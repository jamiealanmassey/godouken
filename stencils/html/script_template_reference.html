Godouken Reference Manual

Godouken Mission Statement

Godouken is a lightweight, non-intrusive and integrated GDScript API documentation tool for all your Godot projects. The aim of the project is to make sharing documentation for your games as simple as possible, increasing the productivity of teams, and/or to serve as a reference for the single programmer.

Godouken Project Settings

There are Godouken specific settings which you can find and access in the `general` tab of the Godot Project Settings dialog!
Include Reference Toggles the generation of this reference manual in your docs
Include Console Suggestions The output of the Godouken console log will make documentation suggestions
Exclude Directories The given list of project directories will be ignored in generation

Godouken Directives

Script Directives
@name This is the desired name of your script
@created The date at which the script was created
@modified The date at which the script was modified
@deprecated The date at which the script was deprecated
@version The current version of the script
@author The author(s) of the script
@brief A brief description of what the script does
@detailed A more detailed description of the script
Method Directives
@brief A brief description of what the method does
@detailed A more detailed description of the method
@parameter[name] Describe each parameter parameter
@return Describe the returned variable
@see Adds hyperlinks to other scripts/methods
@warning Add warning note to this property
@info Add info note to this property
Property Directives
@brief A brief description of what the property is
@detailed A more detailed description of the property
